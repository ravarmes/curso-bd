Práticas 4.2 — DDL: Criação de Tabelas e Índices

Estrutura do CREATE TABLE

CREATE TABLE nome_tabela (
  coluna1 TIPO PRIMARY KEY,
  coluna2 VARCHAR(100) NOT NULL,
  coluna_fk INT,
  FOREIGN KEY (coluna_fk) REFERENCES outra_tabela(pk)
    ON DELETE CASCADE
);

PRIMARY KEY garante unicidade. FOREIGN KEY cria relacionamento. ON DELETE define comportamento ao excluir pai.
