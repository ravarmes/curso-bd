Práticas 6.4 — Plano de Execução e Tuning Básico

EXPLAIN - Plano Estimado

EXPLAIN mostra o plano que o otimizador PRETENDE usar, sem executar a consulta.

Query:

EXPLAIN
SELECT * FROM produto
WHERE nome = 'Notebook';

EXPLAIN ANALYZE - Execução Real

EXPLAIN ANALYZE executa a consulta e mostra tempos reais (não apenas estimados).

Query:

Diagnóstico de Consultas Lentas

Identifique problemas comuns no plano de execução:

⚠️ Seq Scan em tabela grande

Problema: Lendo todas as linhas

Solução: Criar índice na coluna do WHERE

⚠️ Nested Loop com muitas linhas

Problema: JOIN ineficiente

Solução: Criar índice na FK ou usar HASH JOIN

⚠️ Sort externo (disk)

Problema: ORDER BY usando disco

Solução: Aumentar work_mem ou criar índice ordenado

✅ Index Scan

Bom: Usando índice para filtrar

Comandos de Manutenção

VACUUM

VACUUM produto;

Recupera espaço de linhas deletadas/atualizadas. Essencial para performance.

ANALYZE

ANALYZE produto;

Atualiza estatísticas usadas pelo otimizador. Execute após grandes alterações.

VACUUM ANALYZE

VACUUM ANALYZE produto;

Combina ambos. Recomendado para manutenção regular.

REINDEX

REINDEX INDEX idx_nome;

Reconstrói índice corrompido ou inchado.
